# Java DSA Interview Prep

A complete roadmap for mastering Data Structures & Algorithms in Java, with structured study guidance, practice plans, and recommended problem patterns for interview success.

## 🚀 Objective
This repository is designed to help learners move from fundamentals to advanced DSA topics with a proven practice plan. It focuses on:

- Core data structures and algorithms
- Interview patterns and problem-solving strategies
- Consistent practice with real-world examples
- Java-based solutions and implementation details

1. Start with the Fundamentals section and review the concepts.

3. Follow the practice plan and work on progressively harder problems.
4. Use the pattern list to recognize common interview questions.
5. Track progress by topic and revisit weak areas weekly.

## 🔧 Core DSA Techniques

Use these techniques across topics to solve common interview problems:

- Two Pointers: pair elements from both ends, compare, or merge sorted lists.
- Sliding Window: grow/shrink a window over an array or string for max/min subarray problems.
- Fast and Slow Pointers: detect cycles, find midpoints, and compare list halves.
- Prefix Sum / Cumulative Sum: precompute totals to answer range queries quickly.
- Hashing: count, map, and detect duplicates using `HashMap`/`HashSet`.
- Sorting: order input to simplify search, grouping, and pairing problems.
- Binary Search: search sorted data or find boundaries in monotonic functions.
- Divide and Conquer: split a problem into smaller subproblems, such as merge sort.
- Backtracking: build candidate solutions incrementally and prune invalid paths.
- Dynamic Programming: store overlapping subproblem results for optimal substructure.
- Greedy: make locally optimal choices to reach a global optimum.
- Graph Traversal: use BFS/DFS to explore nodes, paths, and connectivity.
- Topological Sort: order tasks with dependencies and detect cycles.
- Union-Find: manage disjoint sets for connectivity and cycle detection.
- Monotonic Stack / Queue: maintain ordered candidates for range queries.
- Bit Manipulation: use bit operations for masks, subsets, and parity checks.
- Window / Interval Techniques: handle interval merging, scheduling, and overlap.
- State Machine / Simulation: model sequential processes and validate state changes.
- Trie / Prefix Tree: solve prefix search and string matching efficiently.
- Segment Tree / Fenwick Tree: answer range queries and updates in logarithmic time.
- Meet-in-the-Middle: split a problem into two halves for exponential search optimization.
- Search on Answer: binary search the solution space when the answer is monotonic.
